1. Signature Verification\n\nEach provider uses different signature methods:\n\n**Stripe (HMAC-SHA256):**\n```\nSignature = HMAC-SHA256(webhook_secret, timestamp.body)\n```\n\n**PayPal (Certificate-based):**\n```\nSignature = HMAC-SHA256(webhook_secret, transmission_id|time|webhook_id|body_hash)\n```\n\n**Chapa (HMAC-SHA256):**\n```\nSignature = HMAC-SHA256(webhook_secret, JSON.stringify(body))\n```\n\n### 2. Idempotent Processing\n\n```\nIdempotency Key = provider + transaction_id\n\nIf webhook already processed:\n  вҶ’ Return immediately (no duplicate processing)\nElse:\n  вҶ’ Enqueue for processing\n```\n\n### 3. Exponential Backoff\n\n```\nRetry 1: 1s\nRetry 2: 2s\nRetry 3: 4s\nRetry 4: 8s\nRetry 5: 16s (max: 5 min)\nAfter 5 failures: Alert admin\n```\n\n### 4. Event Normalization\n\n```\nProvider Event          вҶ’  Normalized Event\nв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ\ncharge.succeeded       вҶ’  payment.success\ncharge.failed          вҶ’  payment.failed\ncharge.refunded        вҶ’  payment.refunded\ncharge.dispute.created вҶ’  payment.disputed\n```\n\n---\n\n## API Usage Examples\n\n###
